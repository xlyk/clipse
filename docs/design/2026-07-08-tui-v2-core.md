# TUI v2 core: follow mode + polish tier

Date: 2026-07-08. Status: approved scope (Kyle). Parent evaluation:
`docs/design/2026-07-08-tui-improvements-draft.md` — this doc fixes the
scope for the first PR; the draft carries the full rationale, mockups, and
per-item detail. Where this doc and the draft disagree, this doc wins.

## Scope (PR 1 — this effort)

From the draft, verbatim by item id:

- **U1 — Follow mode** (owner-requested, build first): `f` on a live row
  opens a full-screen realtime tail of that issue's agent logs;
  `t` toggles between the structured transcript
  (`<board>/logs/<ISSUE>.transcript.jsonl`, rendered semantically with
  lane-colored header, dim one-line tool calls, wrapped assistant text) and
  raw worker stderr (`<board>/logs/<issue>.log`, plain dim). Auto-scroll
  pins to bottom unless the user scrolls up; a `tea.Cmd` polls the active
  file every ~500ms holding a byte offset (I/O never in `Update`);
  a shrinking stderr file means a new spawn `O_TRUNC`ed it — reset offset
  to 0. `esc` returns. Zero dispatcher plumbing: both paths derive from
  `boardDir` in `runTUI`.
- **P1 — Kill the dead middle**: content-size the pipeline panel
  (`min(natural, bodyH − actMin)`), activity absorbs the remainder; drop
  empty-section `· none` lines (empty RUNNING/BLOCKED bands fold into the
  header chips).
- **P2 — One notion of activity**: merge RUNNING + IN FLIGHT into a single
  ACTIVE section (live-claim rows first with spinner/active-lane/elapsed,
  then unclaimed review/rework/merging rows dim with a column label).
  Header chips become `⚡ working · ◇ waiting · • queued · ✖ blocked ·
  ✓ done`.
- **P3 — Two-class activity feed**: verdicts
  (merge/complete/blocked/request_changes/rework_cap_exceeded) bold and
  colored with prose kept; mechanics
  (claimed/promoted/adopted/stale_release/retry_scheduled/orphan*/respawn)
  dim, UUIDs shortened, language translated (`stale` → "claim expired —
  requeued in <col>", `retry_scheduled` → "transient failure — retry
  <n>/<cap> in <backoff>s"). Feed rows gain a lane badge resolved via the
  issue's runs.
- **P4 — Render the seven dropped fields**: `ReworkCount` chip,
  `BlockedUntil` retry countdown, `Unmirrored` badge + header count,
  `HeartbeatAt` stale-heartbeat warning, `Priority` as the QUEUED sort key.
  (`Description` is deferred to detail v2; the seventh field,
  `RecoverAttempts`, rides the rework chip as `⟳ r<n>` when > 0.)
- **P5 — Adaptive palette**: every `lipgloss.Color` constant becomes
  `lipgloss.AdaptiveColor{Light: <github-light analogue>, Dark: <current>}`.
- **P6 — Small fixes batch**: tab/footer label consistency; kanban cards get
  active-lane badge + spinner; DONE line gains PR numbers; footer context
  shows selected row status; cost labeled `est.` and computed with two
  per-lane rates (sonnet-class coder, opus-class reviewer).

## Constraints

- No new Go module dependencies in this PR. bubbles submodules already in
  the module graph are fine.
- Visual direction (draft §3): lane color is identity; verdicts are the
  only loud elements; density over chrome; GitHub-dark stays the dark
  palette.
- House TUI style holds: pure `Update`, I/O only in `tea.Cmd`s, wall-clock
  only in `View`, injected refresh, unit tests without TTY/DB.
- Kernel untouched: `cli/tui/` (+ `cli/tui.go` wiring if needed) only.
- TDD per item where the logic is pure (layout math, feed classification,
  tail-offset handling, chip rendering); `make test` + `make lint` green.

## Acceptance

- All seven items implemented; unit tests cover: pipeline height math
  (tall terminal, 10 cards → activity absorbs remainder), ACTIVE grouping
  (claimed first, unclaimed dim), verdict/mechanic classification + stale
  translation, tailer offset reset on file shrink, transcript-line
  rendering (tool_call/assistant/turn_start), adaptive color table, queued
  priority sort.
- Manual validation: `clipse tui --board` against a kept smoke board
  (`--keep` run or the last run's board dir) — follow mode streams a live
  worker during a `--fast` smoke run.
- `make test` + `make lint` green; draft PR.
